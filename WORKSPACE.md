## 一、新会话启动清单

**Codex 在每次新会话开始后，必须依次执行：**

1. 读取 `D:\workspace\WORKSPACE.md`（本文件）
2. 读取 `D:\workspace\PITFALLS.md`，搜索与当前任务相关的标签
3. 确认当前项目上下文：在哪个项目下工作
4. 继续之前未完成的任务，或开始新任务

## 四、命名规范

### 4.1 项目命名
- **格式**: `Projects-NN-中文说明`（NN 为两位数字序号）
- **正面示例**: `Projects-01-个人助理`、`Projects-05-数据看板`
- **禁止**: 英文名、拼音、含空格、大小写混用

### 4.2 技能命名
- **自定义技能格式**: `阿木-功能名称`
- **正面示例**: `阿木-视频制作`、`阿木-飞书助手`
- **禁止**: 与已有技能重名、使用英文名

### 4.3 文件命名
- 脚本：英文小写 + 下划线（`send_papers.py`、`feishu_api.js`）
- 文档：中文或英文，不含空格（`设计概要.md`、`README.md`）
- 配置：遵循工具默认（`package.json`、`.gitignore`）

### 4.4 Git 分支
- 主分支统一使用 `main`
- 功能分支：`feat-简短描述`
- 修复分支：`fix-简短描述`

## 五、Git 规范

### 5.1 铁律
1. 每个功能点完成后立即提交，禁止堆积
2. 提交前确认 `git status` 干净
3. 禁止提交 
ode_modules/`、`.codex/`、日志、临时文件
4. 每个项目必须有 `.gitignore`

### 5.2 提交格式
```
<类型>: <中文简述>

feat     — 新功能
fix      — 修复 bug
refactor — 重构（不改变功能）
docs     — 文档
chore    — 杂项
```

### 5.3 远程仓库
- 每个项目应在 GitHub 有对应远程仓库
- 仓库名与项目目录名一致
- 推送前确认无敏感信息

## 六、项目创建规范

> 行为规则全部定义在 `AGENTS.md`。

### 6.1 初始化流程
1. 从 `templates/` 复制 .gitignore、README、.codex-project.json
2. `git init && git checkout -b main`
3. 首次提交
4. 建议创建 GitHub 远程
5. 更新 §三 项目索引，标注关联技能
6. 检查 `libs/` 和已有项目是否有可复用代码

### 6.2 共享库
- 可跨项目复用的逻辑放入 `libs/`，做成可导入的包，不复制
- 创建新功能前先查 `libs/`，不重复造轮子
- 技能的 scripts/ 只做编排，核心逻辑放在库或项目中
## 七、Codex 行为契约

### 7.1 代码位置
- 所有代码写入 `D:\workspace\Projects-NN-xxx/`
- 临时脚本放项目内的 `work/` 或 `scripts/`
- 交付文件放项目内的 `outputs/`
- 禁止在根目录创建代码文件

### 7.2 语言
- 对话、注释、文档全部中文
- 代码标识符保留英文

### 7.3 错误处理
- 遇到问题先查 `PITFALLS.md` 是否有记录
- 解决问题后追加到 `PITFALLS.md`
- 不确定立即问用户，不要猜测后继续
